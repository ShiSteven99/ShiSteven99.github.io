Hand Gesture Recognition for Vehicle Controls
Intuitive hand gesture recognition system for in-vehicle controls to reduce distracted driving.
Python MediaPipe Computer Vision
Background

Each year in the US, over 400,000 people get injured and 3,000 people die due to distracted driving. Our team designed an intuitive hand gesture recognition system for internal vehicle controls to reduce distractions on the road. My main contributions were implementing gesture recognition for mode transitions and unifying multiple control functions into a cohesive system architecture allowing for an intuitive user experience.

How It Works

The system uses MediaPipe's hand tracking library to obtain X and Y coordinates through a regular camera, building a library of recognized hand gestures. The design goal was an intuitive gesture set requiring minimal memorization. Only X and Y were used here as we used a simple computer 2D camera for the project which didn't provide a depth map.

System Overview

We prioritized intuitive gesture design and ease of use, reusing gestures across modes where possible to reduce cognitive load—for example, using the same gesture to control both temperature and volume.


  • Media Mode (base) — pause/play music, change songs, control volume with intuitive gestures.
  • Climate Mode — activated by a "1" gesture; control temperature. Auto-returns to Media Mode after 5s of inactivity.
  • Call Mode — activated when a call comes in; accept or decline with a gesture.

Dense Video Captioning with Semantic Alignment
Enhanced PDVC pipeline with semantic alignment achieving 8% improvement across all metrics.
PyTorch CLIP Transformers Deep Learning
Background

Dense video captioning captures and captions different events occurring in a video — with huge implications for video surveillance, autonomous driving, and aiding those with visual impairments. Our team of 4 investigated whether semantically aligning visual and caption features can improve overall performance. My key contributions were developing the Tuner (alignment) architectures and integrating the Tuner with the existing PDVC framework.

Baseline: PDVC Framework

Overview of the baseline PDVC [1] framework

Videos are passed through a pretrained feature extractor → deformable transformer encoder-decoder capturing inter-frame and inter-event interactions → parallel localization head (MLP) and captioning head (LSTM) → event counter.

Semantic Alignment Method

CLIP's [2] frozen text encoder creates caption embeddings. Our Tuner model processes video features, which are projected into the same embedding space as caption features. We then compute a cosine similarity loss between the two, and backpropagate this loss to train the Tuner to better align video representations with their corresponding captions.

The trained tuner is frozen and inserted before the transformer encoder to align visual features with required captions. This pipline is called the modified PDVC.

Results

After training the modified PDVC with the YouCook2 dataset, results were very positive. We found several Tuner architectures to surpass the baseline model. The Conv1 architecture surpassed baseline PDVC in all metrics (BLEU4, METEOR, CIDEr, SODA_c), producing more correct words and more coherent captions.

Few-Shot Fine-Tuning for Comic Generation Pipeline
Text-to-comic pipeline using LLaMA-2 + Stable Diffusion with few-shot style transfer.
Stable Diffusion ControlNet DreamBooth LLaMA-2
Background

We explored a novel approach to comic generation by transforming text prompts into visually expressive comic panels using diffusion models. A pipeline capable of generating comics in specified styles through few-shot fine tuning while maintaining content relevance was developed. Key challenges such as style consistency and controlled style variation were addressed through the implementation of a similarity metric for content preservation. The result is an end-to-end framework that produces coherent, stylistically consistent comic panels from text.

Full Pipeline

LLaMA-2 handles story and prompt generation → A Stable Diffusion based pipeline to generate comic panes → ControlNet with Canny edge maps to transfer styles accross panels.

Two-Stage Diffusion Pipeline

I was responsible for the diffusion pipeline which can be split into three main steps:


  • Stage 1: Generate a realistic images from the text prompt.
  • Stage 2: Apply Canny edge detection to extract structural outlines from the realistic images.
  • Stage 3: Feed edge maps into a ControlNet-guided diffusion model few shot fine-tuned on comic styles using DreamBooth — applying stylistic transfer while preserving composition and structure.
Style Consistency & Fine-Tuning

Using the feature-based style loss calculated via Gram matrices from VGG19 feature maps, we are able to select the best stylistic match from each generated batch. DreamBooth was few shot fine-tuned on ~12 reference general comic style images. The best results came from settings of: guidance scale 7.5 + 60 inference steps.

Maze Navigation Robot
Autonomous robot that traverses a maze, localizes itself, picks up and delivers a block.
Arduino Ultrasonic Sensors Mechatronic Systems SolidWorks
Background

The objective was to build a robot that can traverse through a maze, localize itself, pick up a block, and drop it at a desired location. My main contributions were programming the obstacle avoidance and localization algorithms on the Arduino Uno and Mega, and designing the rack-and-pinion gripper lift mechanism.

System Overview

The project is split into four main tasks:


  • Stage 1: Robot is dropped in the maze at a random location and needs to localize itself
  • Stage 2: Once the robot localizes itself, it needs to traverse to the designated block location and find the block
  • Stage 3: Once the block is located, the robot needs to traverse to it and grasp the block
  • Stage 4: Finally, with the block collected, the robot needs to drop it off at designated location

Through out these 4 stages, the robot needs to be constantly performing obstacle avoidance to not run into walls.

Mechanical Design

Full 3D assembly designed in SolidWorks. A rack-and-pinion mechanism was chosen for the gripper lift for its simplicity and reliable vertical motion. Omni wheels were used to allow for translation motion without turning the vehicle. However this proved to cause many issues due to the poor traction of the wheels.

Final assembled robot

Electronics & Sensors

Electronics used in this project include:


  • Arduino Mega — high-level controller: processes sensor signals and makes decisions.
  • Arduino Uno — low-level motor controller.
  • 4 DC motors (wheels), 2 servo motors (gripper + lift).
  • 7 ultrasonic sensors (S0–S6) for obstacle avoidance and localization.
  • 1 IR sensor (S7) for block detection.
Navigation Algorithm
  • Small-step movement with ultrasonic realignment after each step — solves the omni-wheel slippage issue and ensures robust straight-line driving. Two ultrasonic sensors on the left and right sides of the robot allows for angular alignment with respect to to wall.
  • A right wall following algorithm is used to randomly traverse the maze during localization. The robot will traverse with this until a unique position is identified.
  • Unique positions are identified by sensor readings. There are exactly 4 locations on the map where the robot will see 2 blocks in front and 1 block at the back.
  • For block drop off, there are 4 possible locations, with 3 of them (2-4) as labelled in the below diagram being achievable through right wall following upon leaving pick up zone. Drop off location 1 is programmed in as a separate unique case.
